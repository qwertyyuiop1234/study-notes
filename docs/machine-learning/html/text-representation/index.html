텍스트 표현 (Text Representation)이란?

기계학습 알고리즘은 수치 행렬만 처리할 수 있다. 따라서 인간의 언어를 반드시 수치 객체(numerical objects), 보통은 벡터(vector)로 변환해야 한다. 이 과정을 텍스트 표현(text representation)이라고 하며, 단어 수준의 표현을 워드 임베딩(word embedding)이라 부른다.

📈 발전 흐름: 이산(discrete) 원-핫 차원 → 통계적 분산 표현 + SVD → 예측 기반 Word2Vec → 전역 통계 GloVe → 부분 단어 기반 FastText

1. 이산 표현 & 원-핫 인코딩

Discrete Representation — One-Hot Encoding

📌 개념

말뭉치(corpus)에서 고유 단어로 사전(dictionary/vocabulary)을 구성한다. 각 단어는 사전 크기만큼의 차원을 가지며, 해당 단어의 인덱스만 1, 나머지는 0으로 표현한다.
단어를 벡터로 표현하는 가장 간단한 방법이다.

📖 사전 크기 |V| = 6

[개, 고양이, 늑대, 사자, 송어, 잉어]

개 = [1, 0, 0, 0, 0, 0]

고양이 = [0, 1, 0, 0, 0, 0]

늑대 = [0, 0, 1, 0, 0, 0]

⚠️ 치명적 한계

  1. 1. 대용량 메모리 필요

    • 음성 인식: 20K, 구문 분석: 50K, 대용량 사전: 500K

    • 구글 1T 말뭉치: 13M — 벡터 하나가 1300만 차원!

  2. 2. 유사성 비교 불가능

    서로 다른 원-핫 벡터는 기하학적으로 직교(orthogonal)하므로 내적(dot product)이 항상 0이다.

    $$ \begin{aligned} \text{Sim(개, 늑대)} &= [1,0,0,0,0,0] \cdot [0,0,1,0,0,0]^\mathsf{T} = 0 \\ \text{Sim(개, 잉어)} &= [1,0,0,0,0,0] \cdot [0,0,0,0,0,1]^\mathsf{T} = 0 \end{aligned} $$

    개=늑대=잉어? 😱 의미적 유사성을 전혀 반영하지 못함!

2. 분산 표현 & 공기 행렬

Distributed Representation & Co-Occurrence Matrix

📖 분포 가설 (Distributional Hypothesis)

"비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가질 것이다." 분산 표현은 이 가정에서 출발하여 밀집 벡터(dense vector)로 의미를 인코딩한다.

🦠 의미 유사성 예시 — 코로나 vs 메르스

문장 A: "신종 코로나 바이러스 집단 감염증이 일상생활을 통해 지속 확산되고 있다."

→ 코로나 주변 문맥: {신종, 바이러스, 집단, 감염, 확산, 일상, 생활}

문장 B: "프랑스는 북부도시 릴에서 2명의 메르스 바이러스 감염환자가 발생했다."

→ 메르스 주변 문맥: {바이러스, 감염, 프랑스, 환자, 발생}

✅ Sim(코로나, 메르스) ≠ 0 — 공유 문맥 단어(바이러스, 감염) 덕분에 유사도가 0이 아님!

📊 공기 행렬 (Co-Occurrence Matrix)

선택한 윈도우 크기 내에서 단어들이 함께 등장하는 빈도를 기록한 행렬.

🔤 예문 (Window size = 1)

"I like deep learning. I like NLP. I enjoy flying."

(I, like), (like, deep), (deep, learning), (I, enjoy), (enjoy, flying)

→ Sim(NLP, learning) = ? (아래 코사인 유사도 참고)

📐 코사인 유사도 (Cosine Similarity)

두 벡터 사이의 유사도를 측정하는 척도. 길이로 정규화된 내적을 기반으로 하여 방향성만 비교하므로, 빈도의 절대적 크기에 영향받지 않는다.

$$ \cos(\vec{X},\vec{Y}) = \frac{\vec{X} \cdot \vec{Y}}{\|\vec{X}\|\|\vec{Y}\|} = \frac{\sum_{i=1}^n x_i y_i}{\sqrt{\sum_{i=1}^n x_i^2} \times \sqrt{\sum_{i=1}^n y_i^2}} $$

두 벡터가 동일 방향 → 1, 직각 → 0, 반대 방향 → -1

• 🟢 두 벡터가 동일: cos = 1

• ⚪ 두 벡터가 직각(무관): cos = 0

• 🔴 두 벡터가 반대: cos = -1

💡 Sim(NLP, learning) = 1/2 = 0.5

3. 특이값 분해 (SVD)

Singular Value Decomposition — 차원의 저주 해결

⚠️ 공기 벡터의 문제점 — 차원의 저주

차원이 증가하면서 학습데이터의 수가 차원의 수보다 적어져 성능이 저하되는 현상. 즉 희소 데이터 문제(sparse data problem)를 야기한다. 공기 행렬은 단어 수만큼의 차원을 가지므로 매우 고차원 & 희소하다.

🔢 SVD의 수학적 정의

행렬을 특정한 구조로 분해하는 방식. $n \times m$ 행렬을 세 행렬의 곱으로 분해:

$$ X_{n \times m} = U_{n \times n} \, S_{n \times m} \, V^\mathsf{T}_{m \times m} $$

$U$, $V^\mathsf{T}$는 직교 행렬(orthogonal), $S$는 특이값을 중요도 순으로 정렬한 대각 행렬.

✂️ 절단된 SVD (Truncated SVD)

상위 $k$개의 특이값만 유지하여 잠재 의미 구조(latent semantic structure)를 보존하면서 차원을 축소:

$$ \hat{X}_{n \times m} \approx \hat{U}_{n \times k} \, \hat{S}_{k \times k} \, \hat{V}^\mathsf{T}_{k \times m} $$

작은 노이즈 성분을 제거하고, 지배적인(dominant) 성분만 보존한다.

🐍 Python SVD 예제

import numpy as np

# 예문: "I like deep learning. I like NLP. I enjoy flying."
# Window size 1로 만든 공기 행렬
X = np.array([
    [0, 2, 1, 0, 0, 0, 0, 0],   # I
    [2, 0, 0, 1, 0, 1, 0, 0],   # like
    [1, 0, 0, 0, 0, 0, 1, 0],   # enjoy
    [0, 1, 0, 0, 1, 0, 0, 0],   # deep
    [0, 0, 0, 1, 0, 0, 0, 1],   # learning
    [0, 1, 0, 0, 0, 0, 0, 1],   # NLP
    [0, 0, 1, 0, 0, 0, 0, 1],   # flying
    [0, 0, 0, 0, 1, 1, 1, 0],   # .
])

U, S, Vt = np.linalg.svd(X, full_matrices=False)

# 2차원으로 축소된 word embedding
word_embedding_2d = U[:, :2]
print(word_embedding_2d)

4. SVD → Word2Vec: 예측 기반 워드 임베딩

From SVD To Word2Vec — Prediction-Based Neural Embeddings

❌ SVD（카운트 기반 방법)의 문제점

  • 높은 계산 비용: $n \times m$ 행렬 계산 → $O(mn^2)$ — 말뭉치가 커지면 사실상 불가능
  • 유연성 부족: 새로운 단어나 문서가 추가될 경우 SVD를 처음부터 다시 수행해야 함

💡 핵심 전환: "동시 등장 횟수(count)를 직접 포착하는 대신, 모든 단어의 주변 단어를 예측(predict)하자!"

🧩 CBOW (Continuous Bag-of-Words)

Context → Center

주변 문맥 단어들을 입력으로 받아, 이들의 프로젝션을 합산(average/sum)한 후 중심 단어를 예측한다. 큰 말뭉치에서 빠르게 수렴.

Input: w(t-1), w(t+1) → SUM → Output: w(t)

🎯 Skip-Gram

Center → Context

중심 단어 하나로 주변 여러 문맥 단어들을 예측한다. 하나의 중심 단어가 여러 학습 쌍을 생성하므로 희귀 단어(rare word)에 강함.

Input: w(t) → Output: w(t-1), w(t+1)

⚙️ Word2Vec 작동 원리

1️⃣ 투영 (Projection)

$x_k$: $V$차원 one-hot vector → $N$차원 word embedding으로 축소

$V\text{-dim} \xrightarrow{\text{projection}} N\text{-dim}$

2️⃣ 출력 & 비용 함수

$y_k$: $V$차원 one-hot vector. 목표는 Cross Entropy 최소화.

$$ \text{Loss} = -\log(\hat{y}_i) $$

정답 단어의 로그 확률을 최대화하는 방향으로 학습

5. SoftMax 시간을 단축하는 방법

SoftMax Optimization — |V|만큼의 계산이 필요! 어떻게 해결할까?

SoftMax는 모든 단어 $V$개에 대한 확률을 계산해야 하므로 $O(V)$의 시간이 걸린다. 사전 크기가 수십만~수백만인 경우 현실적으로 불가능. 이를 해결하기 위한 3가지 기법:

🌲 1. Hierarchical SoftMax

트리 구조를 이용한 탐색. 허프만 이진 트리(Huffman binary tree)를 사용하여 복잡도를 $O(V)$에서 $O(\log_2 V)$로 줄인다. 각 노드는 이진 분류기를 학습한다.

🎲 2. Negative Sampling

주변 대표 단어만 샘플링. 하나의 정답(target)과 $K$개의 노이즈(noise) 단어만 사용하여 이진 분류(binary classification) 문제로 변환. 즉, 모든 $V$개 단어를 업데이트하지 않고 일부만!

📉 3. Sub-Sampling

빈번 단어 서브샘플링. "the", "a" 등 과도하게 자주 등장하는 저정보(low-information) 단어를 일정 확률로 드롭하여 학습 속도와 품질을 개선.

6. GloVe: 전역 벡터

Global Vectors for Word Representation — Stanford

🤔 Word2Vec과 GloVe의 가장 큰 차이점

Word2Vec: Prediction 기반

로컬 윈도우 내에서 주변 단어를 예측하는 방식 → 지역적 문맥만 포착

GloVe: Probability 기반

말뭉치 전체의 전역 동시 등장 정보(global co-occurrence)를 활용하면서 단어 벡터를 학습

💡 GloVe의 핵심 통찰

동시 등장 확률의 비율(ratio)이 의미를 인코딩한다!

예: $x=\text{solid}$가 주어졌을 때, $\text{ice}$와 $\text{steam}$의 확률 비율:

$\frac{P(\text{solid}|\text{ice})}{P(\text{solid}|\text{steam})} \gg 1$ → solid는 ice와 관련, steam과는 무관

즉, 확률의 내적(dot product)으로 표현하여 학습 → 내적이 로그 동시 등장 확률과 같아지도록 학습

관계$x=\text{solid}$$x=\text{gas}$$x=\text{water}$$x=\text{fashion}$
$P(x|\text{ice})$큼작음큼작음
$P(x|\text{steam})$작음큼큼작음
$\frac{P(x|\text{ice})}{P(x|\text{steam})}$$\gg 1$$\ll 1$$\approx 1$$\approx 1$

"solid"와 "gas" 같은 변별 단어가 ice와 steam의 의미를 구분해준다. 중립적이거나 무관한 단어는 비율이 1에 가깝다.

🎯 GloVe 목적 함수 (Objective Function)
$$ J = \sum_{i,j=1}^{V} f(X_{ij}) \left( w_i^\mathsf{T} \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2 $$

• $X_{ij}$: 단어 $i$와 $j$의 동시 등장 횟수

• $w_i$, $\tilde{w}_j$: 학습되는 주 단어·문맥 벡터

• $b_i$, $\tilde{b}_j$: 편향(bias) 항

• $f(X_{ij})$: 가중치 함수 — 과도하게 빈번한 쌍의 노이즈 억제

핵심: 내적(dot product)이 로그 동시 등장 확률(log co-occurrence probability)과 같아지도록 fitting

7. FastText: 부분 단어 기반 확장

by Facebook ← Word2Vec by Google

Word2Vec과 GloVe는 각 단어를 독립적인 단위로 취급하므로, 오탈자(misspelling)나 미등록 단어(OOV; Out-of-Vocabulary)에 취약하다. FastText는 문자 수준의 부분 단어 n-gram(subword n-gram)으로 단어를 분해하여 학습한다.

🔑 노이즈에 강한 이유

부분 단어로 학습하므로, 완전히 새로운 단어나 약간 변형된 단어도 공유된 문자 n-gram을 통해 표현할 수 있다. 즉 OOV 문제에 강건(robust)하다.

🔤 "apple"의 3-gram 분해

"<ap" "app" "ppl" "ple" "le>" "<apple>"

최종 단어 벡터는 모든 부분 단어 벡터의 합:

$$ \vec{v}_{\text{apple}} = \sum_{g \in G(\text{apple})} \vec{z}_g $$

$G(\text{apple})$: apple의 모든 n-gram 집합
