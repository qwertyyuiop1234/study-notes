What Is Text Representation?

Machine learning algorithms operate on numerical matrices. Human language must therefore be converted into numerical objects, usually vectors. This process is called text representation, and word-level representations are commonly called word embeddings.

Development flow: text representation evolved from discrete one-hot dimensions, to statistical distributed representation with SVD, to prediction-based Word2Vec, global-statistics-based GloVe, and subword-aware FastText.

1. Discrete Representation and One-Hot Encoding

Concept

A vocabulary is built from unique words in a corpus. One-hot encoding uses the vocabulary size as the vector dimension. Each word receives a unique index; that position is set to 1, and every other position is 0.

Vocabulary $V$ example, size 6

[dog, cat, wolf, lion, trout, carp]

dog = [1, 0, 0, 0, 0, 0]

cat = [0, 1, 0, 0, 0, 0]

wolf = [0, 0, 1, 0, 0, 0]

Critical Limitations

  1. 1. Sparsity and memory waste.

    If a vocabulary has 13,000,000 words, each vector has 13,000,000 dimensions, but only one nonzero entry.

  2. 2. Orthogonality blocks semantic similarity.

    Different one-hot vectors are geometrically perpendicular, so their dot product is always zero.

    $$\text{Sim}(\text{dog}, \text{wolf}) = [1,0,0,0,0,0] \cdot [0,0,1,0,0,0]^T = 0$$

2. Distributed Representation and Co-Occurrence Matrices

Distributional Hypothesis

Words appearing in similar contexts tend to have similar meanings. Distributed representations encode meaning in dense vectors rather than in isolated vocabulary indices.

Context example

corona: [virus, infection, spread, quarantine]

mers: [virus, infection, hospital, patient]

Shared context words allow nonzero similarity in vector space.

Co-Occurrence Matrix

A co-occurrence matrix records how frequently words appear together inside a selected window size.

Sentence, window = 1

"I like deep learning. I like NLP. I enjoy flying."

(I, like), (like, deep), (deep, learning), (I, enjoy), (enjoy, flying)

Cosine Similarity

To reduce distortion from raw frequency magnitude, cosine similarity normalizes by the L2 norms and compares vector direction.

$$\cos(\vec{X},\vec{Y}) = \frac{\vec{X} \cdot \vec{Y}}{\|\vec{X}\|\|\vec{Y}\|} = \frac{\sum_{i=1}^n x_i y_i}{\sqrt{\sum_{i=1}^n x_i^2} \times \sqrt{\sum_{i=1}^n y_i^2}}$$

• Same direction: 1.0

• Orthogonal/unrelated: 0.0

• Opposite direction: -1.0

3. Singular Value Decomposition

Co-occurrence matrices can become high-dimensional and sparse. SVD reduces dimensionality by preserving dominant latent components while removing small noisy components.

Full SVD
$$X_{n \times m} = U_{n \times n} S_{n \times m} V^T_{m \times m}$$

$U$ and $V^T$ are orthogonal matrices; $S$ stores singular values sorted by importance.

Truncated SVD
$$\hat{X}_{n \times m} \approx \hat{U}_{n \times k} \hat{S}_{k \times k} \hat{V}^T_{k \times m}$$

Only the top $k$ singular values are retained to preserve latent semantic structure.

Core NumPy Logic

import numpy as np

X = np.array([[0, 2, 1, 0, 0, 0, 0, 0],
              [2, 0, 0, 1, 0, 1, 0, 0],
              # ...
])

U, S, Vt = np.linalg.svd(X, full_matrices=False)
word_embedding_2d = U[:, :2]

4. Word2Vec: Prediction-Based Neural Embeddings

Problems with SVD count-based methods

  • High initial cost: full matrix factorization can require about $O(mn^2)$ computation.
  • Inflexibility: new words or documents may require rebuilding the matrix and recomputing SVD.

Word2Vec trains embeddings by predicting words inside a sliding context window and updating neural-network weights.

CBOW Model

Context $\rightarrow$ Center

Uses surrounding context words as input, averages or sums their projections, and predicts the center word.

Skip-Gram Model

Center $\rightarrow$ Context

Uses a center word to predict multiple surrounding context words, often producing strong vectors for rare words.

Cross-Entropy Loss Reduction

Because the target $y$ is one-hot, multiclass cross entropy reduces to the negative log probability of the correct word.

$$H(\hat{y}, y) = -\sum_{j=1}^{|V|} y_j \log(\hat{y}_j) \quad \Longrightarrow \quad \text{Loss} = -\log(\hat{y}_i)$$

Softmax Bottleneck Optimizations

1. Hierarchical Softmax

Uses a Huffman binary tree and reduces complexity from $O(V)$ to $O(\log_2 V)$.

2. Negative Sampling

Uses one true target and $K$ noise words, turning learning into a binary classification task.

3. Subsampling

Drops overly frequent low-information words with a certain probability to improve speed and quality.

5. GloVe: Global Vectors for Word Representation

Word2Vec focuses on local prediction windows. GloVe uses global co-occurrence statistics, especially probability ratios, to encode semantic relationships.

Relationship$x=\text{solid}$$x=\text{gas}$$x=\text{water}$$x=\text{fashion}$
$P(x|\text{ice})$largesmalllargesmall
$P(x|\text{steam})$smalllargelargesmall
$\frac{P(x|\text{ice})}{P(x|\text{steam})}$$\gg 1$$\ll 1$$\approx 1$$\approx 1$

Distinguishing words such as “solid” and “gas” separate the meaning of ice and steam; neutral or unrelated words produce ratios close to 1.

Log-bilinear objective
$$J = \sum_{i,j=1}^{V} f(X_{ij}) \left( w_i^T \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2$$

The inner product is fitted to the log co-occurrence count, while $f(X_{ij})$ suppresses overly frequent noisy entries.

6. FastText: Subword-Based Extension

Word2Vec and GloVe usually treat each word as an independent unit, making them vulnerable to misspellings and unseen words. This is the Out-of-Vocabulary (OOV) problem. FastText decomposes words into character-level subword n-grams.

n-gram split for "apple", $n=3$

"<ap""app""ppl""ple""le>""<apple>"

The final word vector is treated as the sum of its subword vector components.

$$\vec{v}_{\text{apple}} = \sum_{g \in G(\text{apple})} \vec{z}_g$$
