Chapter 0

텍스트 표현이란?

기계학습 알고리즘은 수치 행렬만 처리할 수 있다. 따라서 인간의 언어를 반드시 수치 객체(numerical objects), 보통은 벡터(vector)로 변환해야 한다. 이 과정을 텍스트 표현(text representation)이라고 하며, 단어 수준의 표현을 워드 임베딩(word embedding)이라 부른다.

📈 발전 흐름: 이산(discrete) 원-핫 차원 → 통계적 분산 표현 + SVD → 예측 기반 Word2Vec → 전역 통계 GloVe → 부분 단어 기반 FastText
Chapter 1

이산 표현과 원-핫 인코딩

📌 개념

말뭉치(corpus)에서 고유 단어로 사전(vocabulary)을 구성한다. 각 단어는 사전 크기만큼의 차원을 가지며, 해당 단어의 인덱스만 1, 나머지는 0으로 표현한다.

📖 사전 크기 |V| = 6

[개, 고양이, 늑대, 사자, 송어, 잉어]

개 = [1, 0, 0, 0, 0, 0]

고양이 = [0, 1, 0, 0, 0, 0]

늑대 = [0, 0, 1, 0, 0, 0]

⚠️ 치명적 한계

  1. 대용량 메모리 폭증: 사전 크기가 수십만~수백만에 달할 경우 차원이 너무 거대해지고 대부분 0이 채워지는 희소성(sparsity) 문제가 극대화된다.
  2. 유사성 비교 불가능 (직교성): 서로 다른 모든 원-핫 벡터들은 내적(dot product)이 항상 0이다.
    $$ \begin{aligned} \text{Sim(개, 늑대)} &= [1,0,0,0,0,0] \cdot [0,0,1,0,0,0]^\mathsf{T} = 0 \\ \text{Sim(개, 잉어)} &= [1,0,0,0,0,0] \cdot [0,0,0,0,0,1]^\mathsf{T} = 0 \end{aligned} $$

    의미적 유사성(개 ↔ 늑대)을 아예 포착할 수 없습니다!

Chapter 2

분산 표현 & 분포 가설

📖 분포 가설 (Distributional Hypothesis)

"비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가질 것이다." 분산 표현은 이 가정에서 출발하여 고차원 희소 벡터를 저차원 밀집 벡터(dense vector)로 매핑하고 단어 간의 실질 유사도를 측정한다.

🦠 문맥 예시 — 코로나 vs 메르스

A: "신종 코로나 바이러스 집단 감염증이 일상생활을 통해 지속 확산되고 있다."
→ 코로나 문맥: {신종, 바이러스, 감염, 확산}

B: "프랑스는 북부도시 릴에서 2명의 메르스 바이러스 감염환자가 발생했다."
→ 메르스 문맥: {바이러스, 감염, 환자, 발생}

공통 문맥(바이러스, 감염)에 의해 Sim(코로나, 메르스) ≠ 0 달성!

📊 공기 행렬 (Co-Occurrence Matrix)

선택한 윈도우(window) 크기 내에서 단어들이 함께 등장하는 전체 빈도를 기록해 단어를 행 벡터로 표현합니다.

🔤 예문 (Window Size = 1)

"I like deep learning. I like NLP. I enjoy flying."

(I, like), (like, deep), (deep, learning), (I, enjoy), (enjoy, flying)

📐 코사인 유사도 (Cosine Similarity)

두 벡터 사이의 방향성 유사도를 측정합니다. 길이로 정규화된 내적이므로 절대 빈도 크기에 왜곡되지 않는 합리적인 유사도를 제공합니다.

$$ \cos(\vec{X},\vec{Y}) = \frac{\vec{X} \cdot \vec{Y}}{\|\vec{X}\|\|\vec{Y}\|} = \frac{\sum_{i=1}^n x_i y_i}{\sqrt{\sum_{i=1}^n x_i^2} \times \sqrt{\sum_{i=1}^n y_i^2}} $$

동일 방향 → 1 | 직교 무관 → 0 | 정반대 방향 → -1

• cos = 1 : 두 벡터의 의미 방향이 완전히 일치

• cos = 0 : 두 벡터가 서로 완벽히 무관함 (원-핫 직교)

• cos = -1 : 두 벡터의 의미가 완전 정반대 방향

💡 예제 예문 기반: Sim(NLP, learning) = 0.5

2. 수식과 정의

Chapter 3

SVD 차원 축소

공기(co-occurrence) 행렬은 단어 수만큼의 고차원 희소 상태를 유발합니다 (차원의 저주). **특이값 분해(SVD)**를 적용해 주요 성분만 남겨두고 잠재 의미 공간으로 차원을 낮춥니다.

🔢 SVD의 수학적 정의

$n \times m$ 행렬 $X$를 세 직교/대각 행렬의 곱으로 완벽히 분해합니다:

$$ X_{n \times m} = U_{n \times n} \, S_{n \times m} \, V^\mathsf{T}_{m \times m} $$

$U, V^\mathsf{T}$는 직교 행렬, $S$는 중요도 순의 대각 특이값 행렬

✂️ 절단된 SVD (Truncated SVD)

상위 $k$개의 지배적인 특이값만 남기고 잘라내어 잠재 정보만 보존합니다:

$$ \hat{X}_{n \times m} \approx \hat{U}_{n \times k} \, \hat{S}_{k \times k} \, \hat{V}^\mathsf{T}_{k \times m} $$

작은 노이즈 성분은 제거하고 저밀도 의미 구조만 포착

🐍 Python NumPy를 이용한 SVD 예제

import numpy as np

# 예문: "I like deep learning. I like NLP. I enjoy flying."
# 8 x 8 공기 행렬
X = np.array([
    [0, 2, 1, 0, 0, 0, 0, 0],   # I
    [2, 0, 0, 1, 0, 1, 0, 0],   # like
    [1, 0, 0, 0, 0, 0, 1, 0],   # enjoy
    [0, 1, 0, 0, 1, 0, 0, 0],   # deep
    [0, 0, 0, 1, 0, 0, 0, 1],   # learning
    [0, 1, 0, 0, 0, 0, 0, 1],   # NLP
    [0, 0, 1, 0, 0, 0, 0, 1],   # flying
    [0, 0, 0, 0, 1, 1, 1, 0],   # .
])

U, S, Vt = np.linalg.svd(X, full_matrices=False)
word_embedding_2d = U[:, :2] # 2차원으로 축소된 밀집 워드 임베딩
Chapter 4

Word2Vec 예측 모델

❌ 기존 SVD 방식의 명확한 단점
  • 높은 계산 복잡도: 대규모 행렬 연산에 $O(mn^2)$ 비용이 들어 대량 말뭉치 학습이 절대 불가능합니다.
  • 신규 단어 유연성 부재: 새 단어가 하나 추가되면 공기 행렬 전체를 구성하고 SVD를 처음부터 재구동해야 합니다.

이를 극복하기 위해 Mikolov 등이 발표한 Word2Vec (2013)은 직접 동시 등장을 세어 기록하는 것을 그만두고, "신경망 학습을 통해 타겟 단어와 문맥 단어를 직접 예측(predict)하자"는 사상을 정립했습니다.

🧩 CBOW (Continuous Bag-of-Words) Context → Center

주변 여러 문맥 단어들을 받아 프로젝션을 합산(average/sum)한 뒤, 타겟이 될 1개의 중심 단어를 맞춰냅니다. **빠른 연산 수렴 속도**가 특징입니다.

🎯 Skip-Gram Center → Context

중심 단어 1개를 보고 주변의 복수 문맥 단어들을 복합 예측합니다. 1개 중심 단어가 여러 개의 학습 데이터 쌍을 생성하여 **희귀 단어 학습에 뛰어난 강점**을 가집니다.

Chapter 5

Softmax 단축 최적화

전체 사전 크기 $|V|$만큼을 매번 다 합산해야 하는 일반 소프트맥스는 $O(V)$의 무거운 연산 한계를 유발합니다. 이를 극복하는 핵심 3요소입니다.

🌲 1. Hierarchical Softmax

이진 트리(Huffman tree) 구조로 단어를 매핑하여, $|V|$번의 분류 계산을 트리를 타고 내려가는 $O(\log_2 V)$번으로 압축합니다.

🎲 2. Negative Sampling

1개의 실제 정답(Positive)과 몇 개($K$개)의 오답 노이즈(Negative) 샘플 단어들만 모아 시그모이드 이진 분류(binary)로 문제를 간략화합니다.

📉 3. Sub-Sampling

"the", "a" 같이 말뭉치 전체에 수없이 반복되면서 정작 의미 정보가 없는 초고빈도 단어들은 일정 빈도로 임의 유실(drop)시켜 속도를 증진시킵니다.

Chapter 6

GloVe 전역 통계 모델

🤔 Word2Vec vs GloVe의 극단적 대조

Word2Vec은 지정 윈도우 내 단어들을 실시간 예측(Prediction)하므로 말뭉치 전체의 전역 통계를 잡지 못합니다. GloVe(2014)는 **"전역 동시 등장 확률의 비율(ratio)이 단어 간의 기하학적 유사도를 완벽히 보장한다"**는 아이디어를 기반으로 합니다.

💡 GloVe co-occurrence 비율의 마법

$x=\text{solid}$일 때, ice와 steam의 비율:

$\frac{P(\text{solid}|\text{ice})}{P(\text{solid}|\text{steam})} \gg 1$

두 확률의 비율이 단순 예측 확률보다도 훨씬 구체적으로 단어의 의미 관계를 잡아냅니다.

관계 $x=\text{solid}$ $x=\text{gas}$ $x=\text{water}$ $x=\text{fashion}$
$P(x|\text{ice})$ 큼 작음 큼 작음
$P(x|\text{steam})$ 작음 큼 큼 작음
$\frac{P(x|\text{ice})}{P(x|\text{steam})}$ $\gg 1$ $\ll 1$ $\approx 1$ $\approx 1$
🎯 GloVe 목적 함수 (Objective Function)
$$ J = \sum_{i,j=1}^{V} f(X_{ij}) \left( w_i^\mathsf{T} \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2 $$

• $X_{ij}$: 단어 $i, j$의 동시 등장 횟수

• $w_i, \tilde{w}_j$: 주 단어 및 문맥 단어 벡터

• $f(X_{ij})$: 가중치 함수 (고빈도 노이즈 가중치 하향 조절)

두 벡터 내적이 전역 동시 등장 확률과 로그 핏을 맞추도록 훈련합니다.

Chapter 7

FastText 부분 단어 모델

Word2Vec과 GloVe는 각 단어를 더 쪼갤 수 없는 완전체 원자로 취급하기 때문에, 사전에 없었던 오타(misspelling)나 미등록어(OOV)에 대해서는 벡터 표현을 아예 내놓지 못하는 취약점을 안고 있습니다. **FastText (Facebook)**는 단어를 문자 수준 n-gram(subword n-gram)으로 분해하여 훈련합니다.

💡 OOV 문제 극복: 처음 본 신조어나 오타가 들어오더라도, 공유된 서브 3-gram, 4-gram 등의 하위 조각 벡터들의 가중합을 산출하여 합치기 때문에 안정적인 텍스트 특징 값을 제공할 수 있습니다.
🔤 단어 "apple"의 3-gram 분해 결과
"<ap" "app" "ppl" "ple" "le>" "<apple>"
$$ \vec{v}_{\text{apple}} = \sum_{g \in G(\text{apple})} \vec{z}_g $$

최종 벡터는 쪼개진 모든 부분 조각 벡터들의 총 합입니다.
