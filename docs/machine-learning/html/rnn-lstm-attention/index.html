0. 왜 RNN이 필요한가?

Sequential data needs memory

순환 신경망(Recurrent Neural Network, RNN)은 은닉 상태(hidden state)가 시간 방향으로 연결되어, 과거의 요약 정보를 현재 계산에 재사용하는 신경망이다.

자연어 문장, 음성 파형, 주가, 기후 데이터는 한 시점만 독립적으로 보면 정보가 부족하다. “문장을 읽는다”는 것은 현재 단어 $x_t$를 읽으면서 직전까지의 맥락 $h_{t-1}$을 함께 고려하는 과정이다.

핵심 장점
  • 가변 길이 입력: 문장 길이가 달라도 동일한 파라미터를 반복 적용한다.
  • 다양한 입출력 구조: 분류, 생성, 번역, 시퀀스 라벨링 등으로 확장 가능하다.

직관: 긴 글 읽기

책의 320페이지를 이해하려면 319페이지까지 쌓인 사건과 인물 관계를 기억해야 한다. FNN은 매 페이지를 처음 보는 것처럼 처리하지만, RNN은 이전 페이지까지의 요약 메모를 들고 다음 문장을 읽는다.

📌 TL;DR — RNN은 $x_t$만 보지 않고 $h_{t-1}$도 함께 보며, 순차 데이터의 맥락을 hidden state에 압축한다.

1. Feed-forward NN vs Recurrent NN

The single extra term that changes everything

Feed-forward NN

$$h = g(Vx+c)$$ $$\hat{y}=Wh+b$$

시간 개념이 없고 현재 입력 $x$만으로 hidden vector $h$를 만든다.

Recurrent NN

$$h_t = g(Vx_t + Uh_{t-1}+c)$$ $$\hat{y}_t=Wh_t+b$$

이전 상태 $h_{t-1}$가 전이 가중치 $U$를 거쳐 현재 계산에 들어온다.

가중치 공유(Weight Sharing): $V, U, W$는 매 시간 새로 생기는 것이 아니다. 첫 단어와 100번째 단어 모두 같은 파라미터를 사용한다. 그래서 긴 시퀀스에도 같은 규칙을 반복 적용할 수 있고, 파라미터 수가 시간 길이에 비례해 폭증하지 않는다.

2. Folding과 Unfolding

Compact recurrence vs time-expanded computation graph

Folding: 접힌 구조

하나의 recurrent cell이 자기 자신으로 돌아오는 loop를 가진 것처럼 표현한다. 구조를 압축적으로 볼 때 좋다.

$x_t$→RNN cell↺

Unfolding: 펼친 구조

시간 $t=1,2,\dots,n$에 따라 같은 cell을 복사한 것처럼 펼쳐 BPTT(Backpropagation Through Time)를 수행한다.

$h_1$$h_2$$h_3$$h_n$

3. Vanishing / Exploding Gradient

Why vanilla RNN becomes short-term memory

바닐라 RNN은 시간축이 길어질수록 chain rule에서 같은 종류의 Jacobian이 반복적으로 곱해진다. 그래서 멀리 떨어진 과거 시점의 gradient가 현재까지 오기 전에 사라지거나 폭발할 수 있다.

$$h_4 = \phi(W^T\phi(W^T\phi(W^T\phi(W^T h_0 + U^T x_1)+U^T x_2)+U^T x_3)+U^T x_4)$$
Sigmoid

도함수 최대가 $0.25$라서 반복 곱셈 시 gradient가 빠르게 작아진다.

Tanh

sigmoid보다 낫지만 도함수가 대부분 1보다 작아 장기 의존성에는 여전히 취약하다.

ReLU

양수 구간 도함수는 1이지만 전이 행렬 영향으로 값이 발산할 수 있다.

시험 포인트: 활성화 함수를 바꾸는 것만으로는 RNN의 장기 기억 문제가 근본적으로 해결되지 않는다. 반복 곱셈 구조 자체를 완화하는 아키텍처가 필요하고, 그 대표가 LSTM이다.

4. LSTM: 게이트와 Cell State

Long Short-Term Memory

LSTM(Long Short-Term Memory)은 hidden state $h_t$와 별도로 cell state $c_t$라는 장기 메모리 통로를 둔다. Forget/Input/Output gate가 정보의 삭제·쓰기·노출을 조절한다.

$$\begin{pmatrix} i \\ f \\ o \\ g \end{pmatrix}=\begin{pmatrix}\sigma \\ \sigma \\ \sigma \\ \tanh\end{pmatrix}W\begin{pmatrix}h_{t-1} \\ x_t\end{pmatrix}$$ $$c_t=f\odot c_{t-1}+i\odot g \qquad h_t=o\odot \tanh(c_t)$$
Forget gate $f$

과거 cell state를 얼마나 유지할지 결정한다. 0이면 삭제, 1이면 보존.

Input gate $i$ + candidate $g$

새 정보 후보 $g$ 중 어느 성분을 cell state에 쓸지 정한다.

Output gate $o$

cell에 저장된 내용을 hidden state로 얼마나 내보낼지 조절한다.

Trade-off

바닐라 RNN보다 파라미터와 연산량이 커진다. 단순화 대안으로 GRU가 자주 언급된다.

왜 gradient가 더 잘 살아남나? $c_t$의 업데이트는 $f\odot c_{t-1}$와 $i\odot g$의 덧셈 중심 구조를 사용한다. 바닐라 RNN처럼 동일한 전이 행렬이 계속 곱해지는 경로보다 장기 정보 전달에 유리하다.

6. Seq2Seq의 병목과 Attention

From one context vector to selective memory lookup

Seq2Seq는 encoder가 입력 문장을 읽고 하나의 context vector $e$로 압축한 뒤, decoder가 그 벡터를 바탕으로 출력 시퀀스를 생성하는 구조다.

문장이 길어질수록 모든 정보를 단 하나의 벡터에 넣어야 하므로 information bottleneck이 발생한다. Attention은 decoder의 각 출력 시점마다 encoder의 모든 hidden state $h_i$를 다시 조회하고, 현재 decoder state $s_t$와 관련성이 큰 부분에 높은 가중치를 준다.

1. Score
$$score(s_t,h_i)=s_t^T h_i$$

현재 decoder state와 encoder state의 dot product.

2. Softmax
$$\alpha_{t,i}=\frac{e^{score(s_t,h_i)}}{\sum_j e^{score(s_t,h_j)}}$$

점수를 합이 1인 확률 분포로 변환.

3. Context
$$a_t=\sum_i \alpha_{t,i}h_i$$

가중합으로 필요한 정보를 모은다.
