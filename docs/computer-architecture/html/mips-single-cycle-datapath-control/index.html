1. CPU performance begins with three factors

н”„лЎңм„ём„ң м„Өкі„мқҳ м„ұлҠҘмқҖ instruction count, CPI, clock cycle timeмқҳ кіұмңјлЎң мқҪм–ҙм•ј н•ңлӢӨ.

мӢӨн–ү мӢңк°„мқҖ $Execution\ Time = Instruction\ Count \times CPI \times Clock\ Cycle\ Time$ мңјлЎң м •лҰ¬лҗңлӢӨ. ISAмҷҖ compilerлҠ” instruction countм—җ, datapathмҷҖ control м„Өкі„лҠ” CPIмҷҖ clock cycle timeм—җ к°•н•ҳкІҢ мҳҒн–Ҙмқ„ мӨҖлӢӨ.

Single-cycle MIPSлҠ” лӘЁл“  лӘ…л №м–ҙлҘј н•ң clock м•Ҳм—җ лҒқлӮҙлҜҖлЎң CPIлҠ” $1$мқҙлӢӨ. н•ҳм§Җл§Ң clock cycle timeмқҖ к°ҖмһҘ кёҙ лӘ…л №м–ҙ кІҪлЎң, лҢҖн‘ңм ҒмңјлЎң lwмқҳ instruction memory вҶ’ register file вҶ’ ALU вҶ’ data memory вҶ’ register file кІҪлЎңм—җ л§һм¶°м§„лӢӨ.

TL;DR: CPIк°Җ лӮ®м•„лҸ„ clock cycle timeмқҙ кёёл©ҙ м „мІҙ мӢӨн–ү мӢңк°„мқҙ лӮҳл№ м§Ҳ мҲҳ мһҲлӢӨ. Single-cycleмқҖ мқҙн•ҙн•ҳкё° мүҪм§Җл§Ң лӘЁл“  лӘ…л №м–ҙк°Җ мөңмһҘ кІҪлЎңмқҳ л№„мҡ©мқ„ кіөмң н•ңлӢӨ.

2. Logic design: combinational vs. state elements

CPUлҠ” мғҒнғң мҶҢмһҗ мӮ¬мқҙм—җ мЎ°н•© л…јлҰ¬лҘј лҶ“кі , clock edgeл§ҲлӢӨ лӢӨмқҢ мғҒнғңлҘј нҷ•м •н•ҳлҠ” кө¬мЎ°лӢӨ.

Combinational elements

нҳ„мһ¬ мһ…л Ҙл§ҢмңјлЎң м¶ңл Ҙмқҙ кІ°м •лҗңлӢӨ. лӮҙл¶Җ кё°м–өмқҙ м—Ҷкі  propagation delay нӣ„ $Y=f(A,B,\dots)$к°Җ лӮҳмҳЁлӢӨ. мҳҲмӢңлҠ” adder, MUX, ALU, logic gatesмқҙлӢӨ.

State elements

к°’мқ„ кё°м–өн•ңлӢӨ. Program Counter, register file, memoryмІҳлҹј clock edgeм—җ л§һм¶° мғҒнғңлҘј к°ұмӢ н•ңлӢӨ.

Edge-triggered clocking

мғҒнғң мҶҢмһҗлҠ” rising edgeм—җм„ңл§Ң к°’мқ„ л°ӣм•„л“ӨмқёлӢӨ. clock period лҸҷм•Ҳ мЎ°н•© нҡҢлЎңк°Җ м•Ҳм •нҷ”лҗҳкі , лӢӨмқҢ edgeм—җм„ң к·ё кІ°кіјк°Җ м•Ҳм „н•ҳкІҢ м ҖмһҘлҗңлӢӨ. мқҙ к·ңм№ҷмқҙ м—Ҷмңјл©ҙ feedback pathмҷҖ race condition л•Ңл¬ём—җ мғҒнғңк°Җ л¶Ҳм•Ҳм •н•ҙм§„лӢӨ.

Why write control matters

swмҷҖ beqлҠ” register к°’мқ„ ліҙмЎҙн•ҙм•ј н•ҳлҜҖлЎң register fileм—җлҠ” RegWriteк°Җ н•„мҡ”н•ҳлӢӨ. RegWrite=0мқҙл©ҙ write register лІҲнҳёмҷҖ write dataк°Җ мЎҙмһ¬н•ҙлҸ„ registerлҠ” л°”лҖҢм§Җ м•ҠлҠ”лӢӨ.

3. ALU building blocks

MIPS ALUлҠ” AND, OR, full adder, MUXлҘј мЎ°н•©н•ҙ м—¬лҹ¬ м—°мӮ°мқ„ к°ҷмқҖ н•ҳл“ңмӣЁм–ҙлЎң м„ нғқ мӢӨн–үн•ңлӢӨ.

Full adder core

1-bit adderлҠ” $sum=a\oplus b\oplus c_{in}$, $c_{out}=(a\cdot b)+(a\cdot c_{in})+(b\cdot c_{in})$ лЎң н‘ңнҳ„н•  мҲҳ мһҲлӢӨ. 32к°ңмқҳ 1-bit sliceлҘј м—°кІ°н•ҳл©ҙ 32-bit ALUк°Җ лҗңлӢӨ.

Subtraction by reuse

лі„лҸ„ subtractorлҘј л§Ңл“Өкё°ліҙлӢӨ $a-b=a+\bar{b}+1$ м„ұм§Ҳмқ„ мқҙмҡ©н•ңлӢӨ. BinvertлЎң $b$лҘј л°ҳм „н•ҳкі  мөңн•ҳмң„ CarryInм—җ $1$мқ„ л„Јмңјл©ҙ кё°мЎҙ adder chainмқҙ лә„м…Ҳкё°лЎң лҸҷмһ‘н•ңлӢӨ.

slt and beq

  • slt: $a-b$ кІ°кіјмқҳ sign м •ліҙлҘј мөңн•ҳмң„ bitмқҳ Less мһ…л ҘмңјлЎң лҗҳлҸҢл Ө м „мІҙ кІ°кіјлҘј 0 лҳҗлҠ” 1лЎң л§Ңл“ лӢӨ.
  • beq: $a-b=0$мқҙл©ҙ л‘җ к°’мқҙ к°ҷлӢӨ. ALU resultмқҳ лӘЁл“  bitк°Җ 0мқҙл©ҙ Zero=1мқҙ лҗңлӢӨ.

5. Single-cycle datapath flows

лӘ…л №м–ҙ мў…лҘҳл§ҲлӢӨ к°ҷмқҖ datapath мһҗмӣҗмқ„ м„ңлЎң лӢӨлҘё control signal мЎ°н•©мңјлЎң нҶөкіјн•ңлӢӨ.

Instruction Fetch

PC вҶ’ instruction memory вҶ’ 32-bit instruction. лҸҷмӢңм—җ adderк°Җ $PC+4$лҘј л§Ңл“ лӢӨ. MIPS instructionмқҖ 4 byte кі м • кёёмқҙмқҙлҜҖлЎң лӢӨмқҢ мҲңм°Ё мЈјмҶҢлҠ” $PC+4$мқҙлӢӨ.

R-format

rs, rtлҘј мқҪкі  ALUк°Җ funct fieldм—җ л§һлҠ” м—°мӮ°мқ„ мҲҳн–үн•ң л’Ө кІ°кіјлҘј rdм—җ м“ҙлӢӨ.

lw / sw

ALUк°Җ $Reg[rs]+SignExt(offset)$мңјлЎң effective addressлҘј л§Ңл“ лӢӨ. lwлҠ” memory read нӣ„ rtм—җ м“°кі , swлҠ” rt к°’мқ„ memoryм—җ м“ҙлӢӨ.

beq / jump

beqлҠ” subtractлЎң equalityлҘј кІҖмӮ¬н•ҳкі  targetмқҖ $PC+4+(SignExt(offset)<<2)$мқҙлӢӨ. JumpлҠ” 26-bit targetмқ„ shiftн•ҳкі  $PC+4$мқҳ upper bitsмҷҖ кІ°н•©н•ңлӢӨ.

7. Exam pitfalls

  • PC increment: byte addressingмқҙлҜҖлЎң лӢӨмқҢ instructionмқҖ $PC+4$мқҙм§Җ $PC+1$мқҙ м•„лӢҲлӢӨ.
  • Branch base: targetмқҖ нҳ„мһ¬ PCк°Җ м•„лӢҲлқј $PC+4$ кё°мӨҖмңјлЎң кі„мӮ°н•ңлӢӨ.
  • Immediate: offsetмқҖ sign extension нӣ„ ALUлӮҳ branch adderм—җ л“Өм–ҙк°„лӢӨ.
  • sw: register write-backмқҙ м—ҶлӢӨ. RegWrite=0.
  • beq: ALU addк°Җ м•„лӢҲлқј subtractлЎң equalityлҘј кІҖмӮ¬н•ңлӢӨ.
  • Don't care: XлҠ” мӢ нҳёк°Җ м—ҶлӢӨлҠ” лң»мқҙ м•„лӢҲлқј кІ°кіјм—җ мҳҒн–Ҙмқ„ мЈјм§Җ м•ҠлҠ”лӢӨлҠ” лң»мқҙлӢӨ.
