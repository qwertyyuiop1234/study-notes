1. Three classes of concurrency problems

Use this classification first before choosing a synchronization tool.

2. Deadlock: the four necessary conditions

Deadlock can occur only when all four conditions hold simultaneously. Prevention breaks at least one.

3. Trylock and livelock simulator

Two threads may avoid deadlock by releasing and retrying, but if they keep retrying in sync, no one progresses.

T1: lock(L1), trylock(L2)
T2: lock(L2), trylock(L1)

    4. Prevention strategies mapped to conditions

    The exam-level answer is not just naming the four conditions; it is connecting each condition to a design move and its trade-off.

    Break circular wait

    Impose a global or partial lock ordering. Example: always acquire L1 before L2.

    Break hold-and-wait

    Acquire all required locks in one atomic acquisition phase. Simpler reasoning, less concurrency.

    Relax no preemption

    Use trylock, release what you hold on failure, then retry. Beware livelock and cleanup.

    Reduce mutual exclusion

    Use lock-free atomic primitives such as compare-and-swap when appropriate.

    5. Compare-and-swap intuition

    CAS succeeds only if the memory location still contains the value you expected. Otherwise, another thread changed it, so retry with the new state.

    do {
    old_head = head;
    new_node->next = old_head;
} while (CompareAndSwap(&head, old_head, new_node) == 0);
