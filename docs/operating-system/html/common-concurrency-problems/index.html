0. 전체 지도: 이 노트에서 잡아야 할 것

세마포어와 뮤텍스를 활용하여 상호 배제와 순서 제어를 구현하더라도, 개발자의 미흡한 락 설계나 흐름 판단 누락으로 인해 실제 시스템에는 다양한 동시성 오류가 발생합니다. 이 노트의 목적은 동시성 버그를 체계적으로 분류하고, 발생 조건과 수학적 회피 공식을 학습하여 결함 없는 멀티스레드 코드를 도출하는 것입니다.

먼저 이해할 것

  • 뮤텍스 및 세마포어 API의 기본 연산 방식
  • 공유 변수 접근 경쟁 상황(Race Condition)
  • OS 스레드 context switch 메커니즘

최종적으로 할 수 있어야 할 것

  • Atomicity Violation과 Order Violation의 명확한 차이 증명
  • 데드락이 성립하는 4대 필수 불가결한 하드웨어/소프트웨어 조건 나열
  • CAS(Compare-And-Swap) 기반의 Lock-free 연산식 해석

2. 수식과 정의

데드락이나 락 오버헤드를 근본적으로 해결하기 위해 하드웨어가 제공하는 atomic 원자적 기화 연산인 **Compare-And-Swap(CAS)**의 논리입니다.

$$\text{CAS}(\text{address}, \text{expected}, \text{new}) \implies \begin{cases} \text{write new to address and return true}, & \text{if } *\text{address} == \text{expected} \\ \text{return false}, & \text{otherwise} \end{cases}$$

이를 기반으로 락 없이 경쟁을 회피하여 값을 더하는 Lock-free 어셈블리 큐 패턴입니다.

void add(int value) {
    do {
        int expected = counter;
    } while (!CompareAndSwap(&counter, expected, expected + value));
}

3. 직관과 그림

소프트웨어의 결합도가 올라갈수록 락 획득 순서가 모듈 깊이 속에 숨어버려 데드락이 발생하기 쉬워집니다.

의존성 캡슐화에 의해 하위 시스템 간의 락 획득 순서가 꼬이는 원형 대기 그래프의 전형적인 모습.

4. 예제

1) Atomicity Violation (원자성 위반) 예제

// Thread 1
if (proc_info) {
    fputs(proc_info, ...);
}

// Thread 2
proc_info = NULL;

T1이 조건 점검 후 실행하기 직전 선점(preempt)당해 T2가 NULL로 지우면 NULL Pointer Dereference 폭발이 일어남.

2) Order Violation (순서 위반) 예제

// Thread 1
void init() {
    worker = createThread(workerMain);
    initialized = true;
}

// Thread 2
void workerMain() {
    state = worker->state;
}

T1이 initialized 플래그를 미처 1로 켜기도 전에 스케줄러가 T2를 먼저 가동시키면 초기화되지 않은 구조체를 사용해 크래시 유발.

5. 자주 틀리는 부분

오해 1

동시성 버그는 전부 데드락이다? 아닙니다. 통계적으로 70%에 육박하는 버그들이 순서 위반이나 원자성 위반 같은 비데드락(Non-deadlock) 형태입니다.

오해 2

스레드 생성(create)만 하면 생성 즉시 동기화가 맞춰진다? 아닙니다. 스케줄러는 비결정적이기 때문에 락이나 CV로 순서(happens-before)를 강제해야 합니다.

오해 3

CAS는 CPU 부하가 전혀 없다? 아닙니다. 지속적으로 do-while 스핀을 돌기 때문에 지속적인 경쟁 시 CPU 낭비가 발생합니다.

시험 함정: 데드락 예방을 하겠다며 모든 락 획득 시 pthread_mutex_trylock() 만을 난발하여 루프를 돌리면, 두 스레드가 서로 락을 풀었다 쥐었다를 무한 반복하는 **Livelock (라이브락)**의 수렁에 빠져 CPU 이용률만 100%로 낭비하게 됩니다.

반드시 대답할 수 있어야 하는 것

  • 데드락 발생 4가지 필수조건 (상호배제, 점유대기, 비선점, 순환대기)

비교해서 구분할 것

  • Deadlock Prevention (예방 - 자원 제약으로 발생 방지)
  • Deadlock Avoidance (회피 - 동적 세이프 상태 모니터링)
  • Deadlock Detection (탐지 - 주기적 주기 그래프 순환 점검)
구두시험 답변 템플릿: "동시성 오류는 원자성 위반, 순서 위반, 교착 상태로 나뉘며, 교착 상태는 4대 성립 조건 중 순환 대기(Circular Wait)를 락 획득의 엄격한 선형적 순서화(Lock Ordering)를 통해 격파함으로써 예방할 수 있습니다."
